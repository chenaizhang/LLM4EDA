## 模块名称
riscv_xilinx_2r1w (Xilinx Dual-Read Single-Write Memory)

## 逻辑类型
时序逻辑（子模块）

注意：作为子模块，其 process() 调用由顶层模块（riscv_regfile）每个周期触发，代表一个时钟上升沿。
不需要 clk 输入端口和时钟边沿检测逻辑。

## 功能概述
riscv_xilinx_2r1w 实现了 Xilinx FPGA 的 RAM16X1D 原语仿真，提供双端口读、单端口写的存储器功能。每个 32 位寄存器使用 32 个 RAM16X1D 原语实现，构成 16 条目 x 32 位的存储阵列。

## 端口定义

### 输入信号
| 端口名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| ra_i | 3:0 | input | 读端口 A 地址 |
| rb_i | 3:0 | input | 读端口 B 地址 |
| rd0_i | 3:0 | input | 写端口地址 |
| rd0_value_i | 31:0 | input | 写数据 |

### 输出信号
| 端口名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| ra_value_o | 31:0 | output | 读端口 A 数据 |
| rb_value_o | 31:0 | output | 读端口 B 数据 |

## 功能说明

### 1. RAM16X1D 原语仿真
- 每个 RAM16X1D 实现 16 个地址深度、1 位宽度的双读单写存储
- 32 个 RAM16X1D 并行构成 32 位数据宽度
- 写操作同步（时钟上升沿），读操作异步（组合逻辑）

### 2. 地址空间
- 4 位地址（16 个条目）
- 每个条目 32 位宽
- 支持两个独立的读端口和一个写端口

### 3. 读写行为
- process() 调用时：先读取 ra_i、rb_i 地址处的数据（读优先于写）
- 写操作：将 rd0_value_i 写入 rd0_i 地址
- 读操作返回的数据是写入前的旧值（写后读不转发）
