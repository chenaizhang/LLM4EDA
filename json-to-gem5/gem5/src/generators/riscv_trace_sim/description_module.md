## 模块名称
riscv_trace_sim (RISC-V Trace/Simulation Monitor)

## 逻辑类型
时序逻辑（子模块）

注意：作为子模块，其 process() 调用由顶层模块（riscv_issue / riscv_pipe_ctrl）每个周期触发。
不需要 clk 输入端口和时钟边沿检测逻辑。

## 功能概述
riscv_trace_sim 是 RISC-V 处理器的跟踪/仿真监测模块，用于记录每个流水线阶段处理的指令信息（PC 和指令码）。它主要用于仿真调试，记录指令的踪迹信息。

## 端口定义

### 输入信号
| 端口名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| valid_i | 1 | input | 指令有效标志 |
| pc_i | 31:0 | input | 程序计数器值 |
| opcode_i | 31:0 | input | 指令操作码 |

### 输出信号
无输出端口（仅记录内部状态）

## 功能说明

### 1. 指令跟踪记录
- 每个周期当 valid_i 有效时，记录 pc_i 和 opcode_i 到仿真跟踪缓冲区
- 用于流水线调试和功能验证

### 2. 状态输出
- 通过内部状态（波形或日志）记录指令流
- 无直接输出端口，但内部寄存器可用于调试观察
